1 先给你要的总结：大家用 Obsidian 主要在做什么

1.1 1）个人知识管理（PKM）/ 第二大脑

最常见。

大家拿它做：

  • 读书笔记
  • 课程笔记
  • 文章摘录
  • 知识卡片
  • 永久笔记 / 原子笔记
  • 主题知识库

常见方法：

  • PARA
  • Zettelkasten
  • MOC（Map of Content）
  • 双链笔记 / 标签体系 / Dataview 汇总

核心价值：

  • 不是“存文档”，而是把零散信息变成可链接、可复用的知识网络。

1.2 2）日记 / Daily Notes / 生活记录

非常高频。

大家会记：

  • 每日日志
  • 灵感
  • 会议纪要
  • 复盘
  • 情绪 / 健身 / 睡眠 / 阅读 / 开销

常见玩法：

  • Daily Notes 做当天入口
  • 周报 / 月报自动汇总
  • 用 Tasks / Dataview 拉出“本周完成了什么”
  • 用模板固定每天的记录结构

核心价值：

  • 低门槛记录 + 后续可检索、可回顾。

1.3 3）任务管理 / GTD / 项目推进

很多人把 Obsidian 当轻量项目管理器。

大家会管：

  • 待办
  • 项目状态
  • 会议行动项
  • 跟进清单
  • OKR / 周计划

常见插件组合：

  • Tasks
  • Dataview
  • Templater
  • Periodic Notes / Calendar
  • Kanban

典型做法：

  • 笔记里顺手写任务
  • 用查询把所有 #todo / - [ ] 聚合到一个总看板
  • 按项目、截止日期、上下文分类查看

核心价值：

  • 任务和资料在一起，不像单独待办工具那样上下文割裂。

1.4 4）写作工作台

很多人用它写：

  • 博客
  • 公众号草稿
  • 小说
  • 长文研究
  • 课程大纲
  • 演讲稿

常见玩法：

  • 大纲 → 卡片 → 初稿 → 定稿
  • 用双链把资料、灵感、引用、段落关联起来
  • 用 Canvas / 白板梳理结构
  • 最后发布到 Hexo / Hugo / Quartz / MkDocs / GitHub Pages

核心价值：

  • 从素材积累到正式写作是一条链。

1.5 5）学术 / 研究 / 文献管理

学生、研究者、写论文的人用得很多。

大家会做：

  • 文献阅读卡
  • 研究主题梳理
  • 论文大纲
  • 术语概念网络
  • 引用与摘录整理

常见组合：

  • Zotero + Obsidian
  • 文献元数据同步
  • 按论文 / 主题 / 方法论组织
  • Dataview 生成阅读清单

核心价值：

  • 把“看过很多论文”变成“真的形成自己的研究结构”。

1.6 6）工作知识库 / SOP / 会议系统

职场用途也很常见。

大家拿来做：

  • 会议纪要
  • SOP
  • 客户资料
  • 项目文档
  • 产品需求梳理
  • 复盘库

典型方式：

  • 每个项目一个主页
  • 每次会议一条笔记
  • 所有会议自动挂到项目页下
  • 任务从纪要里抽出来统一汇总

核心价值：

  • 文档、会议、行动项在一个系统里闭环。

1.7 7）人生管理 / 目标追踪 / Habit Tracker

很多人把它当“人生仪表盘”。

会记：

  • 年目标
  • 月计划
  • 习惯打卡
  • 财务记录
  • 健身记录
  • 情绪记录
  • 旅行计划

常见插件：

  • Tracker
  • Dataview
  • Calendar
  • Excalidraw / Canvas

核心价值：

  • 把抽象目标变成可追踪记录。

1.8 8）第二层玩法：把 Obsidian 做成“个人操作系统”

这是更进阶的一类。

他们不是只记笔记，而是把 Obsidian 用成：

  • 个人主页 / Dashboard
  • 项目中控台
  • 知识图谱入口
  • 任务中心
  • 每日工作台
  • 个人 CRM
  • 内容生产流水线

常见特征：

  • 首页就像控制台
  • 有大量 Dataview 查询块
  • 模板自动创建标准页面
  • 多种插件联动
  • 笔记即数据库

2 我看到的共性结论

2.1 Obsidian 最受欢迎的，不是“记笔记”

而是这 4 件事叠加：

  • 记录
  • 链接
  • 查询
  • 复用

它强在：

  1. 文本本地化，长期可控
  2. 双链和标签很适合知识网络
  3. 插件生态强，能拼出自己的工作流
  4. 既能简单用，也能玩很深

它的短板也很明确：

  1. 上手门槛比普通笔记高
  2. 很容易折腾插件，掉进“搭系统不做事”
  3. 协作能力不如 Notion / 飞书文档
  4. 移动端能用，但复杂工作流体验一般

适合：

  • 愿意投入搭建成本的人

7 设置循环（重复）任务

在 Obsidian 的 Tasks 插件中，设置循环（重复）任务的语法非常灵活，核心就是使用 🔁 这个 Emoji 加上自然语言描述。

以下是具体的语法规则和常用写法：

7.1 🔑 核心语法

基本格式： 🔁 加上你的重复规则
（注意：🔁 和规则文字之间要有一个空格）

7.2 📝 常用循环规则示例

Tasks 插件支持非常接近日常口语的“自然语言”来定义时间，以下是几种常见场景：

  • 按天/周/月/年重复：
    • 🔁 every day（每天）
    • 🔁 every week（每周）
    • 🔁 every month（每月）
    • 🔁 every year（每年）
  • 指定星期几重复：
    • 🔁 every Monday（每周一）
    • 🔁 every Friday（每周五）
    • 🔁 every Monday, Wednesday, Friday（每周一、三、五）
  • 按间隔重复：
    • 🔁 every 2 weeks（每两周）
    • 🔁 every 3 days（每3天）
    • 🔁 every 4 weeks on Friday（每隔4周的周五）
  • 相对时间重复（基于完成时间）：
    • 🔁 every week when done（每周重复，但下一次任务的日期会根据你实际勾选完成的时间来推算，而不是原定日期）

7.3 💡 完整任务写法示例

一个完整的循环任务通常还会搭配截止日期（📅）等符号。例如：

  • 写周报（每周五）：
    - [ ] 写周报 🔁 every Friday 📅 2026-05-15
  • 缴房租（每月1号）：
    - [ ] 缴房租 🔁 every month on the 1st 📅 2026-06-01
  • 团队会议（每两周的周一）：
    - [ ] 团队同步会议 🔁 every 2 weeks on Monday 📅 2026-05-25

7.4 ⚙️ 循环任务的工作机制

当你勾选完成一个带有 🔁 规则的任务时，Tasks 插件会自动执行以下操作：

  1. 将当前任务标记为完成（变成 - [x]，并自动加上完成日期 ✅）。
  2. 自动在该任务的下方（或上方，取决于设置）生成一条全新的、未完成的任务。
  3. 新任务的日期会根据你设定的规则自动更新（例如今天是5月13日，设定了 every day，新任务的日期就会自动变成5月14日）。

7.5 🛠️ 偷懒小技巧

如果你觉得手写这些 Emoji 和规则很麻烦，可以使用 Tasks 插件自带的快捷命令：

  • 快捷键： 将光标放在任务行，按下 Ctrl + Shift + K（Mac 为 Cmd + Shift + K），会弹出一个图形化窗口。
  • 弹窗设置： 在弹窗的 Recurs 一栏中，直接输入 every week 等规则即可，插件会自动帮你生成对应的语法。
