解决了什么痛点？

  1. 云端笔记库，数据在后台不属于你，分散存储，使用和分析不方便；
  2. 本地笔记软件电脑和手机端同步难，随时随地记录不方便；
  3. 构建知识库时AI如何接入你的笔记库。

设计初衷

极致的通用

用HTML文件驱动，不像markdown还得用特定软件打开渲染，几乎任何终端都有浏览器都能识别，所见即所得。除了环境和依赖，系统只有800K，你可以把他装在电脑、手机、云服务器的任何地方。需要用的时候，随时可以将articles目录下的笔记用优盘拷走或者线上发送，放入目录就能用。单独的HTML不依赖于笔记系统也可以发给任何AI帮你修改。

极致的简约

Markdown 作为嵌入源码保存，每个 HTML 文件底部都保留 Markdown 源码。浏览器看到的是 HTML，编辑器拿到的是 Markdown。 HTML for humen， Markdown for AI，这种方式，你和AI都能一目了然。

极致的开放

超级酷炫的页面是能实现的，你可以让任何AI做成的任何HTML文件，放进articles目录。你也可以写个大概，然后丢给AI，让他美化返回个HTML文件，3D动画、炫酷特效、数据图表都可以实现，这很简单。

极致的克制

尽量保留知识库所需的记录和管理信息、辅助个人日常任务的核心功能，并将这些功能便利性尽量做到极致。提供的主要功能：日常笔记、 网络文章收录、内部双链、图片和附件上传和管理、历史版本、回收站，还有每日任务、全站笔记中的待办事项、提醒事项。笔记与系统任务可以互相联动，你不需要把所有任务都写在“工作中心”页面里。普通笔记里只要按规范写待办 / 提醒标记，系统就能自动读取出来，显示到工作中心。因为真实使用中，任务本来就应该长在上下文里。

提供必要的安全性和便利性

如设置网站密码、提供分享链接、提供标题模糊以便于演示时隐藏敏感信息、提供手机、电脑端和云端的安装指引。
